# 모델은 왜 여러 속성을 리턴할 수 없나

## 상황

회의 중 어느분이 이런 이야기를 했다. "상품 추천 모델을 학습 시킬때 상품의 모든 속성을 가지고 학습을 하니 모델이 리턴할때 상품 ID랑 이름을 같이 리턴하면 되지 왜 ID만 리턴해서 또 DB를 조회하느냐". 평소에 아무런 의심을 안했는데 듣고보니 맞는 말이다. 그럼 왜 모델은 ID랑 이름을 같이 리턴하지 못하고 ID만 리턴했을까.

## 1개만 리턴하는 이유

머신러닝 모델은 **수많은 데이터 속에서 규칙을 찾아내어, 우리가 궁금해하는 특정한 목표 값(Target Feature) 하나를 정확하게 찍어주는 장치**이다. 즉, 모델 자체가 1개만 리턴할 수 있다. 머신러닝을 공부하지 않았다면 납득하기 어렵겠지만 원래 모델이 그렇다.

## 추천 말고 다른 머신러닝 모델은 무엇을 리턴하는가

추천 모델은 추천되는 대상의 ID를 리턴하는것 까지는 이해하였다. 그럼 다른 모델은 무엇을 리턴할까?

### 1. 스팸 메일 분류기 (분류)

가장 대표적인 예시이다. 수많은 이메일 텍스트가 모델에 입력되지만, 모델이 내뱉는 정답은 단 하나입니다.

- 입력(Input): 메일 제목, 보낸 사람, "광고", "당첨" 같은 단어들의 집합
- 타겟(Target): "스팸(1)" 또는 "정상(0)"

### 2. 중고차 가격 예측 (회귀)

- 입력(Input): 주행거리, 연식, 사고 유무, 브랜드명
- 타겟(Target): "1,500만 원"

### 3. 내일의 날씨 기온 예측 (회귀)

- 입력(Input): 오늘 습도, 풍향, 구름의 양, 어제의 기온
- 타겟(Target): "25도"

## ID만 리턴했을때 장점

이제 모델이 1개의 결과만 리턴하는건 납득?하였다. 그러면 다시 추천 모델을 생각해 보자. 만약 정말로 모델이 ID랑 이름을 같이 리턴하면 좋을까?  
만약 도서 추천 모델이 ID와 책 이름을 같이 리턴 했다고 가정하자. 그런데 학습 데이터에 "무소유"를 잘못입력해서 "풀소유"라고 입력했다고 해보자. 그러면 모델도 다시 학습시켜야 한다. 따라서 변경될 여지가 있는 속성은 DB로 조회하는데 데이터 최산화 관점에서도 도움이 된다.
