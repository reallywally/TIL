# 도구

- 도구는 에이전트가 원는 결과를 달성하기 위해 수행할 수 있는 특정 기능 또는 일련의 작업을 말한다.

## 랭체인 기초

- 도구라는게 어떻게 사용되는지 이해하기 위해 초간단 랭체인을 살펴본다.

```python
from langchain.chat_models import init_chat_model
from langchain_core.tools import tool

@tool
def add(x: float, y: float) -> float:
    """ x와 y를 더한다"""
    return x + y

llm = init_chat_model(model="gpt-5-mini", temperature=0)
llm_with_tools = llm.bind_tools([add])
```

### 로컬 도구

- 로컬에서 실행하는 도구로 만들고 수정하기가 쉽다.
- 대표적인 예로 산술 연산, 시간대 변환 등이 있다.
- 명확한 함수 이름을 사용해야 LLM이 적절한 상황에서 도구를 사용한다.
- 로컬에서만 사용하다 보니 공통 라이브러리 처럼 관리해야될때 배포, 유지보수가 어렵다.

### API 기반 도구

- 에이전트가 API를 이용하여 외부 서비스와 상호작용할 수 있다.
- 대표적인 예로 날씨 정보, 주식 정보, 번역 등이 있다.
- API는 항상 성공하는게 아니기 때문에 예외처리를 잘 해야 한다.

### 플러그인 도구

- 책에서 말하는 플러그인 도구는 확장 프로그램(익스텐션) 정도 수준같다.
- 대표적인 예로 ChatGPT에 실시간 검색기능, 클로드는 도구 사용, 제미나이는 vertex AI API를 설명한다.

### MCP

- 모델에 통합 표준을 제공해 LLM과 외부 시스템을 연결하는 통일된 방식의 인터페이스이다.
- 비유하면 제조사 마다 다른 충전기 케이블을 쓰다가 USB C타입으로 통일되는것과 같다.

### 도구의 상태 유지

- LLM이 로컬 도구, API 도구, MCP 등 다양한 서비스를 사용하기에 상태 유지는 중요하다.
- 실제로 지속적으로 유지된 상태를 이용하여 DB 최적화를 한다며 데이터를 지운 경우도 있다.
- 이런 위험을 줄이기 위해 최소 권한 원칙으로 설계해야 한다.

## 자율 도구 생성 / 도구 자가 개발

- 에이전트가 사람이 준 도구만 쓰는 게 아니라, 직면한 문제를 해결하기 위해 필요한 코드나 기능을 스스로 설계하고 빌드한다.

### 파운데이션 모델을 활용한 도구 개발

- 에이전트가 코드를 분석해서 헬퍼 함수, API 명세서, 테스트 코드 등을 만들 수 있다.
- 개발시간은 단축할 수 있으나 검증은 개발자의 감독이 필요하다.

### 실시간 코드 생성

- 깃헙의 코파일럿이 대표적인 예이다.
- 역시 품질, 보안 재현성, 리소스 등 증요한 고려사항이 있다.

## 도구 사용 설정

- 보통 파운데이션 모델 API는 tool-choice라는 파라미터를 통해 도구 사용 방식을 제어할 수 있다.
  - auto: 도구 호출 여부를 스스로 결정
  - any 또는 required: 최소 1개의 도구를 반드시 호출하도록 강제
  - none: 모든 도구 호출 차단
- 올바른 도구를 사용했는지 확인하는것 또한 중요하다.
  - 스키마 검증: jsonschema, Pydantic으로 누락된 필드 도는 잘못된 구조 파악
  - 지능적 재시도: 일시적 오류를 특정 횟수 만큼 다시 하거나 오류 발생시 전체를 다시 하는게 아니라 문제된 부분 부터 다시 시작
  - 안정적 폴백: 재시도가 실패하면 백업 모델을 사용하거나 사용자에게 추가 입력 요청
  - 로깅: 모든 단계를 기록
