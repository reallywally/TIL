# 7. 에이전틱 시스템의 학습

- 그동한 학습한 내용을 에이전틱 시스템을 만드는 것으로 적용해 보자

## 7.1 비모수적 학습

### 7.1.1 비모수적 예시 학슴

- 가장 단순한 기법이 예시 학습(exempler learning)이다.
- 예제가 많다면 프롬프트에 추가할 수도 있겠지만 결국엔 비용과 지연시간을 증가시키다.
- 그래서 가장 관련성 높은 예제를 프롬프트에 포함하는 방법이 일반적인 해결 방법이다.

### 7.1.2 리플렉시온

- 언어 기반 간단한 자기 비판 습관을 부여한다.
- 시간이 지나면서 성찰 내용은 메모리 버퍼에 저장되서 새로운 시도를 시작하기 전에 최근 성찰을 다시보면서 모델 재학습을 하지 않고 결과를 개선할 수 있다.
- 간단한 코드 몇줄로 성능을 상당히 높일 수 있다.

### 7.1.3 경험 학습

- 미모수적 학습을 한 단계 더 확장한 개념이다.
- 에이전트가 경험을 DB에 쌓고 쌓은 데이터를 기반으로 미래의 정책을 개선하는 추가 단계를 적용한다.

## 7.2 모수적 학습: 파인튜닝

- 특정 작업에서의 성능을 개선하기 위해 파라미터를 조정하는 방법이다.
- 파인튜닝 자체는 매우 어렵기 때문에 이번 장에서는 이런 개념이 있다 정도만 이해하고 넘어가자.

### 7.2.1 대형 파운데이션 모델 파인튜닝

- GPT, 클로드, 제미나이 등 모델을 파인 튜닝하는 것이다.
- 파인튜닝을 고려해볼만한 경우
  - 도메인 특화가 중요한 경우
  - 일관된 톤과 형식이 중요한 경우
  - 도구, API 호출이 매우 정확해야하는 경우
  - 충분한 고품질 데이터와 예산이 있는 경우
  - 재학습 주기를 감당할 수 있는 경우
- 파인튜닝을 미루는것이 좋은 경우
  - 빠른 프로토타이핑 단계이나 사용량이 적은 경우
  - 모델의 진화 속도가 투자 대비 비효율적인 경우
  - 자원이 제약되는 경우
- 어설프게 할바에는 안하는게 나음

### 7.2.2 소형 모델

- 파운데이션 모델 대비 레이어와 파라미터가 적어 의사결정 과정을 분석하고 출력에 영향을 미치는 요인을 찾기 쉬움
- 특히 설명 가능성이 필요한 금융, 헬스케어 등에서 유용할 수 있음
- 비용과 접근성도 좋음

### 7.2.3 SFT(Supervised fine tuning): 지도 파인튜닝

- 어떻게 응답해야 하는지 명시적인 예제로 보여 줌으로써 에이전트의 행동을 정밀하게 조정하는 기본 접근 방식

### 7.2.4 DPO(Direct Perference Optimization): 직접 선호 최적화

- 결과를 사람이 선택하여 출력이 사람의 품질 순위 판단에 더 가깝게 정렬되도록 한다.

### 7.2.5 RLVR(Reinforcement Learning with Verifable Rewards): 검증 가능 보상 강화 학슴

- 선호 학습과 가치 기반 정책 최적화를 결합함으로써 강화 파인튜닝을 확장한한다.
- 무슨말인지 잘 모르겠음
