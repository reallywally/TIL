# chapter6 지식과 메모리

- 메모리를 통해 에이전트는 여러 턴과 세션에 걸쳐 연속성을 유지하며 과거 상호작용을 기억한다.

## 6.1 메모리 기본 사용법

### 6.1.1 컨텍스트 윈도우 관리

- 컨텍스트 윈도우란 모델을 호출할때 파운데이션 모델에 함께 제공되는 모든 정보를 의미한다.
- 가장 단순한 메모리 접근법이다.
- 컨텍스트 윈도우가 가득차면 가장 오래된 컨텍스트 부터 제거된다. aka FIFO
- 그래서 관련성이 높거나 중요한것과는 상관 없이 어느 순간 정보가 사라질 수 있는 단점이 있다.

### 6.1.2 전체 텍스트 검색

- 전체 텍스트를 검색하지만 순서대로 조회가 아니라 역색인을 하여 전체를 스캔하지 않고 빠르게 조회할 수 있다.
- 대부분 BM25 스코어링 함수를 사용한다.

## 6.2 시멘틱 메모리와 벡터 스토어

### 6.2.1 시맨틱 검색

- 기존 키워드 검색과 달리 컨텍스트와 의도를 이해하는 것을 목표로하여 더 정확하고 의미 있는 검색 결과를 제공한다.
- 핵심은 **의미**에 초점을 맞춪것이고 이에 기반이 되는 기술이 **임베딩**이다.

### 6.2.2 벡터 스토어로 시맨틱 메모리 구현

- 고차원 벡터 형태를 저장하는 DB가 벡터 DB이다.
  - 예) FAISS, Annoy 등
- 일반 RDB에서 조건으로 검색하듯 벡터 DB도 관련성이 높은 임베딩 값들을 조회한다.

### 6.2.3 RAG

- 검색 기반 기법과 생성 모델을 결합해 에이전틱 시스템의 능력을 강화하는 방법
- 모델이 학습하지 않은 데이터를 사용할 수 있다는 가장 큰 단점
- 퍼플렉시티가 대표적

### 6.2.4 시맨틱 경험 메모리

- 시맨틱 스토어와 외부 지식 DB를 통합하는것은 외부 지식을 주입하는데 효과적이다.
- 하지만 에이전트가 매 세션마다 백지 상태로 시작하거나 컨텍스트 윈도우 제약이라는 단점이 있다.
- 이를 보완하기 위해 시맨틱 경험 메모리가 생겼다.

## 6.3  그래프 RAG

- 기존 RAG가 어려움을 겪는 상황
  - 답이 여러 문서에 흩어져 있는 정보를 연결해야 할때
  - 쿼리가 데이터 셋 전체에 걸친 상위 수준의 의미론적 주제를 요약하기를 요구할때
  - 데이터 셋이 크고 정리가 덜 되었거나 서사형 구조로 구성되었을때
- 이런 문제를 해결하기 위해 그래프 RAG가 생김

### 6.3.1 지식 그래프 활용(Using Knowledge Graphs)

- 프로덕션 레벨로 사용가능한 neo4j-graphrag-python, 교육이나 로컬 실험으로 사용 가능한 nano-graphrag, example-graphrag 등 패키지가 잘 되어저 있다.
- 그래프 RAG 구성 요소
  - 지식 그래프: 데이터를 그래프 형식으로 저장하며 엔티티(노드)와 이들의 관계(엣지)가 명시적으로 정의
  - 검색 시스템: 그래프 DB를 사용한다
  - 생성 모델
- 큰 규모의 데이터 위에서 추론해야 하는데 기존 RAG가 한계에 부딪힌다면 그래프 RAG가 비싸지고 복잡하지만 더 나은 결과를 내는 경우가 많다.
