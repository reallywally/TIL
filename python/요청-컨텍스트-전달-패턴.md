# 요청 컨텍스트 전달 패턴

## 상황

router에서 받은 request body를 A service → B service → C service 까지 넘겨야 할 때, 파라미터로 계속 전달하는 것 외에 다른 방법을 고민했다.

## 권장 방법

### 1. Context 객체 만들어서 넘기기

DTO 같은 Context 객체를 사용하는 방법이다. 가장 간단한 해결 방법이다.

```python
# context.py
from pydantic import BaseModel

class RequestContext(BaseModel):
    user_id: int
    order_data: OrderCreate
    
    class Config:
        arbitrary_types_allowed = True

# router
@router.post("/orders")
async def create_order(body: OrderCreate, request: Request):
    request.state.ctx = RequestContext(
        user_id=body.user_id,
        order_data=body
    )
    await order_service.process(request)
```

### 2. FastAPI의 Depends 사용

의존성 주입을 사용하는 방법이다. FastAPI를 사용한다면 이 방법이 좀 더 FastAPI 스러운 코드 같다.

```python
# 서비스가 직접 ctx를 주입받음
@router.post("/orders")
async def create_order(ctx: RequestContext = Depends(set_context)):
    await order_service.process(ctx)

# order_service.py
async def process(ctx: RequestContext):
    await payment_service.charge(ctx)
```

### 3. contextvars 사용

Python 표준 라이브러리의 `contextvars`를 사용하는 방법이다. 글로벌 변수처럼 "어디서든 꺼내 쓸 수 있는" 편의성을 유지하면서, 비동기 컨텍스트 스위칭 시 값이 섞이는 문제를 해결한다.

```python
# context.py
from contextvars import ContextVar

request_context: ContextVar[RequestContext] = ContextVar("request_context")

# middleware 또는 router
@router.post("/orders")
async def create_order(body: OrderCreate):
    token = request_context.set(RequestContext(
        user_id=body.user_id,
        order_data=body
    ))
    try:
        await order_service.process()
    finally:
        request_context.reset(token)  # 반드시 reset

# order_service.py — 파라미터 없이 꺼내 씀
async def process():
    ctx = request_context.get()
    await payment_service.charge()
```

### 어떤 방법이 권장될까

처음엔 3번 contextvars가 좋다고 생각했다. 스프링에서는 ```TheadLocal```을 사용했었고 글로벌 컨텍스트를 사용하면 어디서든 꺼내 쓸 수 있어 편한데 매번 DTO 같은 클래스를 만들고 함수에 파라미터로 추가하는게 불편하다 생각했다.  
하지만 contextvars는 데이터 흐름은 명시적으로 보이지 않는다. 만약 router에서 contextvars로 저장하고 A, B 서비스를 거처 C 서비스에서 값을 꺼냈다면 나중에 이 코드를 본 사람은 어디서 이 값을 저장했는지 부터 파악해야 한다. 즉 비즈니스 로직을 파악하는데 힘들어 진다. 각각의 장단점이 있지만 가능하면 1, 2번처럼 데이터 흐름이 잘 보이는 방법이 권장된다.

## 해결은 할 수 있지만 권장하지 않는 방법

좋은 방법이 아니더라도 어떤 방법이 있고 왜 좋지 않은지 알면 좋으니 알아보자.

### 1. global 변수 / context 저장

FastAPI는 async 동시 처리 환경이다. 단일 이벤트 루프 안에서도 비동기 컨텍스트 스위칭이 발생하기 때문에 아래처럼 사용하면 의도치 않게 값이 변경될 수 있다.

```python
class GlobalContext:
    user_id: int = None
    order_data: dict = None

ctx = GlobalContext()  # 모듈 레벨 싱글톤

# 요청 A: ctx.user_id = 1 세팅
# 요청 B: ctx.user_id = 2 세팅  ← A의 값이 덮어씌워짐
# 요청 A: ctx.user_id 읽으면 2가 나옴 💥
```

### 2. request 객체 통으로 넘기기

사실 이건 긴 설명과 예제 코드도 필요 없이 사용해서는 절대 안되는 방법이다.
