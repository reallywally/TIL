# 프롬프트 메모

요즘IT에서 클로드 코드로 3일만에 개발하여 위기를 극복한 사례로 작성된 글을 읽었다. 무엇보다 회사의 거래가 끊기고 손해배상까지 이어질 수 있는 심각한 상황에서도 새로 개발하여 문제를 해결하려는 자세가 정말 대단하다고 느껴젔다. 그리고 실제로 작성된 프롬프트를 보니 나중에 활용하기도 좋을것 같아서 메모해둔다.

### 시나리오

```markdown
# 콩콩프레스 실시간 투표 - MVP 사용자 시나리오

## 🎯 MVP 목표

익명 사용자가 태블릿에서 실시간 투표에 참여할 수 있는 **최소 기능 제품(MVP)** 구현.

### 핵심 기능

- 익명 로그인 없는 투표 참여
- 실시간 투표 결과 표시
- 관리자 기본 세션 제어

## 사용자 시나리오

### 📱 참가자 플로우 (MVP)

1. **접속**: 태블릿으로 웹페이지 접속 → UUID 자동 생성
2. **대기**: SSE 연결 후 대기 화면
3. **투표**: 질문 화면에서 선택지 클릭
4. **진행**: 관리자가 다음 질문 활성화
5. **완료**: 감사 메시지 표시

### 관리자 사전 준비 플로우

#### 1. **투표 항목 생성** (투표 CRUD)

- 투표 관리 대시보드에서 새 투표 생성
- 질문 텍스트와 선택지 입력
- 선택지별 색상 설정
- 저장하여 투표 풀에 보관

#### 2. **세션 생성** (세션 CRUD)

- 세션 관리 페이지에서 새 세션 생성
- 세션 제목, 설명, 이벤트 연결
- 투표 취소/재투표 허용 여부 설정

#### 3. **투표와 세션 연결**

- 기존 투표 목록에서 원하는 투표들 선택
- 세션에 투표 추가 및 순서 배치
- 질문별 시간 제한 설정 (선택사항)

#### 4. **화면 준비**

- 오퍼레이션 화면: 관리자 제어요 (노트북/태블릿)
- 프리젠테이션 화면: 참가자 보기용 (대형 스크린)
- 두 화면 동기화 확인

### 🧑‍💼 관리자 실시간 플로우 (MVP)

1. **세션 시작**: 투표 활성화
2. **실시간 모니터링**: 참여자 수 및 응답 현황 확인
3. **질문 이동**: 이전/다음 질문으로 순차 진행
4. **결과 확인**: 실시간 투표 결과 차트 보기
5. **세션 종료**: 투표 종료

---

## 📘 MVP 시나리오

### 시나리오 1: 기본 실시간 투표

- **상황**: 컨퍼런스 세션 만족도 조사
- **참여자**: 100명

**진행**

1. 관리자가 5개 질문 세션 생성
2. 참가자들이 태블릿으로 접속 → UUID 자동 생성
3. 관리자가 투표 시작 → 첫 질문 표시
4. 참가자들이 선택지 클릭하여 투표
5. 실시간 결과가 관리자 화면에 표시
6. 관리자가 다음 질문으로 진행
7. 5개 질문 완료 후 종료 메시지 표시

---

### 시나리오 2: 세션 관리 및 네비게이션

- **상황**: 워크숍 피드백 수집
- **참여자**: 50명

**진행**

1. 관리자가 기존 세션을 복사하여 새 세션 생성
2. 질문 내용 일부 수정
3. 투표 시작 후 2번째 질문에서 참여율 저조 발견
4. 1번째 질문으로 다시 돌아가 재투표
5. 3번째 질문으로 건너뛰어 핵심 질문 먼저 진행
6. 완료 후 결과 데이터 확인

---

### 시나리오 3: 투표 데이터 관리

- **상황**: 제품 발표 후 피드백 정리
- **참여자**: 200명

**진행**

1. 투표 완료 후 관리자가 투표 관리 대시보드 접속
2. 전체 투표 목록에서 응답 수 확인
3. 의심스러운 응답(1초 내 응답) 발견
4. 해당 투표를 검토 후 수동으로 판단
5. 결과 투표를 선택하여 상세 확인
6. 최종 정리된 데이터를 CSV로 내보내기

---

### 시나리오 4: 관리자 사전 준비 전체 과정

- **상황**: 분기별 임직원 만족도 조사 준비
- **목표**: 이벤트 시작 전 모든 투표 시스템 준비 완료

#### 사전 준비 과정

1. **투표 항목 생성 (투표 CRUD)**
   - 투표 관리 대시보드 접속
   - "업무 만족도" 투표 생성: 5개 선택지
   - "복지/보상 평가" 투표 생성: 4개 선택지
   - "업무환경 개선점" 투표 생성: 6개 선택지
   - 총 3개 투표 항목을 투표 풀에 저장

2. **세션 생성 (세션 CRUD)**
   - 세션 관리 페이지에서 "2024 Q1 임직원 만족도 조사" 세션 생성
   - 이벤트: 2024 전사 워크숍에 연결
   - 설정: 투표 취소/재투표 허용, 결과 실시간 표시

3. **투표 세션 연결**
   - 세션 편집 모드에서 기존 투표 추가 버튼 클릭
   - 투표 순서 설정
     1. 업무 만족도
     2. 복지/보상
     3. 업무환경
   - 각 질문별 제한 시간 30초 설정

---

## ⚠️ 예외 상황 (MVP)

### 연결 문제

- SSE 연결 끊김 → 자동 재연결 (3회 시도)
- 세션 종료 후 접속 → "투표가 종료되었습니다" 안내

### 투표 오류

- 중복 투표 → "이미 투표하셨습니다" 메시지 표시
- 관리자 투표 수정/삭제 시 실시간 반영

---

## 🎯 MVP 성능 목표

### 기본 요구사항

- 동시 접속자 수: 최대 300명
- 응답 시간: 투표 반영 2초 이내
- 페이지 로딩: 5초 이내
- 브라우저: Chrome, Safari 지원

---

## 🛠 기술 스택

- **백엔드**: Go + Echo + GORM
- **실시간 통신**: SSE + Redis Pub/Sub
- **프론트엔드**: Vue3 + Vite
- **데이터베이스**: MySQL
```
