# 윈도우의 루트는 어디인가

## 상황

지금까지는 항상 같은 팀 개발자가 모두 맥을 쓰거나 윈도우를 쓰거나 통일되게 사용했다. 그런데 나는 윈도우를 쓰는데 다른 개발자가 맥을 사용하게 되었다.
그런데 파일 업로드 경로를 설정하려 yml을 수정하다보니 궁금한 내용이 생겼다. 맥은 경로를 루트 경로가 /로 명확한데 윈도우는 루트가 C 드라이브로 표시되고 실제 디렉토리도 전부 c:/data 이렇게 사용한다. 그렇다면 yml에 경로는 어떻게 적어야 할까? 윈도우와 맥용 yml을 따로 만들어야 할까?

## 결론

다행히 똑똑한 스프링 덕분에 업로드 경로를 /data/images만 하여 c드라이브에 잘 업로드 되었다. 즉, 윈도우의 루트는 c:/이고 루트 경로는 그냥 /로 하여도 문제가 없다.
